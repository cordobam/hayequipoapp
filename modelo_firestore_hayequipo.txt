===========================================================
1. RESUMEN DE LA APP
===========================================================

App para que un grupo de jugadores pueda:
- Organizar partidos de distintos deportes (Fútbol 5, Tenis,
  Paddle, Basquet, etc.)
- Armar grupos de amigos fijos para invitar rápido
- Invitar a esos grupos y recibir confirmaciones
- Buscar jugadores en la app cuando falta gente
- Llevar estadísticas de los partidos jugados
- Tener un directorio de jugadores con sus datos y preferencias
- Reseñar jugadores post-partido (visibilidad todo/nada)
- Encontrar y gestionar sedes/canchas donde se juega


===========================================================
2. COLECCIONES FIRESTORE
===========================================================

  sports/              → Deportes disponibles en la app
  venues/              → Sedes/canchas donde se juega
  players/             → Perfiles de jugadores/usuarios
  friend_groups/       → Grupos de amigos para invitar rápido
  matches/             → Partidos programados y jugados
  match_invitations/   → Invitaciones a partidos (todo el flujo)
  match_stats/         → Estadísticas por partido (por jugador)
  player_stats/        → Estadísticas agregadas por jugador/deporte
  player_reviews/      → Reseñas post-partido entre jugadores

--- 3.1 Sport ---

Nombre en Firestore: sports/{sportId}

Propósito: Catálogo de deportes. Define cuántos jugadores,
cómo se anota, duración, etc.

data class Sport(
    val id: String = "",
    val name: String = "",              // "Fútbol 5"
    val description: String = "",
    val type: String = "team",          // "team" | "individual" | "pairs"
    val playersPerTeam: Int = 5,
    val minPlayersPerTeam: Int = 4,
    val maxPlayersPerTeam: Int = 7,
    val teamCount: Int = 2,
    val scoringUnit: String = "goles",  // "goles" | "puntos" | "sets"
    val durationMinutes: Int = 40,
    val iconUrl: String = "",
    val isActive: Boolean = true
)


--- 3.2 Venue ---

Nombre en Firestore: venues/{venueId}

Propósito: Sedes deportivas donde se juegan los partidos.
Incluye ubicación (GeoPoint), horarios, servicios, etc.

data class Venue(
    val id: String = "",
    val name: String = "",              // "Club Deportivo X"
    val address: String = "",
    val location: GeoPoint? = null,     // para búsqueda geográfica
    val phone: String = "",
    val photos: List<String> = emptyList(),
    val sportIds: List<String> = emptyList(), // deportes disponibles ahí
    val pricePerHour: Double = 0.0,
    val amenities: List<String> = emptyList(), // "estacionamiento", "duchas"
    val rating: Double = 0.0,
    val schedules: Map<String, ScheduleDay> = emptyMap(),
    val createdBy: String = ""          // playerId del creador
)

data class ScheduleDay(
    val open: String = "08:00",
    val close: String = "23:00",
    val isOpen: Boolean = true
)


--- 3.3 Player ---

Nombre en Firestore: players/{playerId}

Propósito: Perfil del jugador/usuario. Vinculado a Firebase Auth.

data class Player(
    val id: String = "",
    val uid: String = "",               // Firebase Auth UID
    val name: String = "",
    val email: String = "",
    val phone: String = "",
    val photoUrl: String = "",
    val preferredSports: List<String> = emptyList(), // sportIds
    val preferredVenues: List<String> = emptyList(), // venueIds
    val friendGroupIds: List<String> = emptyList(),  // grupos a los que pertenece
    val position: String = "",          // "delantero", "base", etc.
    val skillLevel: Int = 3,            // 1 a 5
    val isAvailable: Boolean = true,
    val showReviews: Boolean = true,    // todo/nada: muestra reseñas en su perfil
    val fcmToken: String = "",          // para notificaciones push
    val createdAt: Timestamp? = null
)


--- 3.4 FriendGroup ---

Nombre en Firestore: friend_groups/{groupId}

Propósito: Grupo reutilizable de amigos para armar partidos
rápido sin tener que invitar uno por uno.

data class FriendGroup(
    val id: String = "",
    val name: String = "",              // "Los Pibes", "La Liga"
    val createdBy: String = "",         // playerId del creador
    val memberIds: List<String> = emptyList(), // playerIds
    val sportIds: List<String> = emptyList(), // deportes que juega el grupo
    val createdAt: Timestamp? = null
)


--- 3.5 Match ---

Nombre en Firestore: matches/{matchId}

Propósito: Representa un partido (programado, en curso o finalizado).

data class Match(
    val id: String = "",
    val sportId: String = "",
    val venueId: String = "",
    val organizerId: String = "",
    val groupId: String? = null,        // grupo de amigos que originó el partido
    val title: String = "",
    val description: String = "",
    val date: Timestamp? = null,
    val durationMinutes: Int = 60,
    val status: String = "scheduled",
        // "scheduled" → "confirmed" → "in_progress"
        // → "finished" | "cancelled"
    val pricePerPlayer: Double = 0.0,   // costo dividido entre jugadores
    val teams: List<MatchTeam> = emptyList(),
    val playersNeeded: Int = 0,
    val createdAt: Timestamp? = null,
    val updatedAt: Timestamp? = null
)

data class MatchTeam(
    val name: String = "",              // "Local", "Visitante"
    val playerIds: List<String> = emptyList(),
    val score: Int? = null
)


--- 3.6 MatchInvitation ---

Nombre en Firestore: match_invitations/{invitationId}

Propósito: Flujo completo de invitación a un partido:
invitar → aceptar/rechazar → confirmar asistencia → asistió/no-show.

data class MatchInvitation(
    val id: String = "",
    val matchId: String = "",
    val groupId: String? = null,        // si vino de una invitación grupal
    val playerId: String = "",
    val invitedBy: String = "",         // playerId del organizador
    val status: String = "pending",
        // "pending"       → invitado, esperando respuesta
        // "accepted"      → dijo que va
        // "rejected"      → no puede ir
        // "confirmed"     → confirmado pre-partido
        // "attended"      → asistió
        // "no_show"       → no apareció
    val isFromSearch: Boolean = false,  // si vino de búsqueda pública
    val respondedAt: Timestamp? = null,
    val createdAt: Timestamp? = null
)


--- 3.7 MatchStat ---

Nombre en Firestore: match_stats/{matchId}_{playerId}

Propósito: Estadísticas de un jugador en un partido específico.

data class MatchStat(
    val id: String = "",                // composite: matchId_playerId
    val matchId: String = "",
    val playerId: String = "",
    val teamIndex: Int = 0,
    val stats: Map<String, Int> = emptyMap(),
        // {"goals": 2, "assists": 1, "fouls": 3}
        // El map es flexible para adaptarse a cada deporte
    val rating: Double = 0.0,
    val createdAt: Timestamp? = null
)


--- 3.8 PlayerStat ---

Nombre en Firestore: player_stats/{playerId}_{sportId}

Propósito: Estadísticas agregadas de un jugador en un deporte,
incluyendo métricas de reseñas.

data class PlayerStat(
    val id: String = "",                // composite: playerId_sportId
    val playerId: String = "",
    val sportId: String = "",
    val matchesPlayed: Int = 0,
    val matchesWon: Int = 0,
    val matchesLost: Int = 0,
    val goals: Int = 0,
    val assists: Int = 0,
    val extraStats: Map<String, Int> = emptyMap(),
    // Métricas de reseñas
    val averageReliability: Double = 0.0,   // promedio 1-5
    val totalReviews: Int = 0,
    val wouldPlayAgainCount: Int = 0,
    val updatedAt: Timestamp? = null
)


--- 3.9 PlayerReview ---

Nombre en Firestore: player_reviews/{reviewId}

Propósito: Reseña cruzada post-partido entre jugadores.
Todos reseñan a todos. La visibilidad es todo/nada por jugador
(flag showReviews en Player).

data class PlayerReview(
    val id: String = "",
    val matchId: String = "",
    val reviewerId: String = "",        // quién reseña
    val reviewedId: String = "",        // a quién reseñan
    val reliability: Int = 5,           // 1-5 (asistió/se copó/compromiso)
    val skill: Int = 3,                 // 1-5 nivel de juego
    val attitude: Int = 5,              // 1-5 actitud
    val comment: String = "",           // "Siempre se copa y viene"
    val wouldPlayAgain: Boolean = true,
    val createdAt: Timestamp? = null
)

5. SECURITY RULES (Firestore)
===========================================================

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Deportes: solo lectura pública autenticada
    match /sports/{sport} {
      allow read: if request.auth != null;
      allow write: if request.auth != null;
    }

    // Sedes: el creador puede editar, todos leen
    match /venues/{venue} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
      allow update, delete: if request.auth.uid
                             == resource.data.createdBy;
    }

    // Jugadores: cada uno edita su propio perfil
    match /players/{player} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
      allow update, delete: if request.auth.uid
                             == resource.data.uid;
    }

    // Grupos de amigos: el creador administra
    match /friend_groups/{group} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
      allow update, delete: if request.auth.uid
                             == resource.data.createdBy;
    }

    // Partidos: el organizador administra
    match /matches/{match} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
      allow update, delete: if request.auth.uid
                             == resource.data.organizerId;
    }

    // Invitaciones: el invitado ve y responde las propias
    match /match_invitations/{invitation} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
      allow update: if request.auth.uid
                    == resource.data.playerId;
      allow delete: if request.auth.uid
                    == resource.data.invitedBy;
    }

    // Estadísticas de partido
    match /match_stats/{stat} {
      allow read: if request.auth != null;
      allow write: if request.auth != null;
    }

    // Estadísticas agregadas: solo lectura
    match /player_stats/{stat} {
      allow read: if request.auth != null;
      allow write: if false; // solo se actualiza desde backend
    }

    // Reseñas: todos pueden crear, el reseñado puede borrar
    match /player_reviews/{review} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
      allow delete: if request.auth.uid
                    == resource.data.reviewedId;
    }
  }
}

6. CONSULTAS TÍPICAS QUE SOPORTA EL MODELO
===========================================================

  1. Partidos disponibles para un deporte
     → matches.whereEqualTo("sportId", sportId)
              .whereEqualTo("status", "scheduled")
              .whereGreaterThan("playersNeeded", 0)

  2. Próximos partidos de un jugador
     → matches.whereArrayContainsAny("teams.playerIds",
                                      playerId)
              .whereGreaterThan("date", Timestamp.now())

  3. Invitaciones pendientes de un jugador
     → match_invitations.whereEqualTo("playerId", playerId)
                        .whereEqualTo("status", "pending")

  4. Perfil de un jugador
     → players.document(playerId).get()

  5. Reseñas visibles de un jugador
     → player_reviews.whereEqualTo("reviewedId", playerId)
     → filtrar en cliente o función cloud según showReviews

  6. Estadísticas de un jugador en un deporte
     → player_stats.document("${playerId}_${sportId}").get()

  7. Buscar jugadores por deporte y disponibilidad
     → players.whereArrayContains("preferredSports", sportId)
              .whereEqualTo("isAvailable", true)

  8. Sedes cerca (requiere librería geo query)
     → venues.whereNear("location", center, radiusKm)

  9. Sedes que soportan un deporte
     → venues.whereArrayContains("sportIds", sportId)

  10. Historial de partidos de un jugador
      → match_stats.whereEqualTo("playerId", playerId)
                   .orderBy("createdAt", descending)

  11. Miembros de un grupo de amigos
      → players.whereArrayContains("friendGroupIds", groupId)


===========================================================
7. FLUJO COMPLETO DE INVITACIÓN Y RESEÑAS
===========================================================

Paso 1: Crear grupo de amigos
  → friend_groups (name, memberIds, sportIds)
  → Cada miembro lo ve en su Player.friendGroupIds

Paso 2: Crear partido
  → matches (sportId, venueId, date, groupId, teams...)
  → Si tiene groupId, se invita automáticamente a los miembros

Paso 3: Invitar al grupo o a jugadores sueltos
  → match_invitations para cada playerId (status=pending)
  → Llega notificación FCM: "¿Jugamos el sábado?"

Paso 4: Responder invitación
  → El jugador acepta (status=accepted) o rechaza (status=rejected)

Paso 5: Si falta gente → buscar en la app
  → Filtrar players por preferredSports, isAvailable, skillLevel
  → Invitar jugadores de búsqueda (isFromSearch=true)

Paso 6: Pre-partido → confirmar asistencias
  → match_invitations.status = "confirmed"

Paso 7: Post-partido
  → match_invitations.status = "attended" | "no_show"
  → Se crean player_reviews cruzados entre todos los que asistieron
  → Se actualizan player_stats (matchesPlayed, averageReliability...)

Paso 8: Reseñas públicas
  → Cada jugador decide si las muestra (Player.showReviews)
